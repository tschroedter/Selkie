The WPF Prototype

I used the WPF prototype to confirm the result of the Ant Colonization Algorithm (ACO). There is no fancy UI, just the bare minimum to setup ACO and show the results. The main window is split up into two sections: the left hand side allows configuring the system and the right hand side shows the survey lines.

The left hand side contains:

  • A control panel to start and stop the ACO computing.
  • A settings panel to change the minimum turn radius and allowed turns.
  • A history panel showing the found solutions and statistics.
  • A pheromones panel showing min, max and average values including a grey scale image.

The right hand side contains:

  • A kind of ‘map’ showing the lines to be surveyed and the racetracks to get from one line to the next. The ‘map’ will be updated whenever a new best trail/path is found.

Both turns allowed

The image below shows the result for a set of lines which are 60 points apart on a 45 degree angle using the default setting (turn radius 60 points and both turns allowed).

  • The green highlighted node indicates the start point of the track.
  • The red highlighted node indicates the end point of the track.

The history table shows Iteration, length, delta, delta %, type and trail:

  • Iteration is the number of iterations used to find the track.
  • Length is the calculated length of the track from start to end including racetracks.
  • Delta relative to the first found track.
  • Delta in percent relative to the first found track.
  • Type is the ant type which found the track.
  • Trail is a list of nodes defining the order in which the survey lines should be traveled.
